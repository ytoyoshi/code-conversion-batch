=== アプリケーションの責務

本アプリケーションは、指定された入力ファイルを読み込み、パラメータで指定された変換元コードタイプ（Java の Charset名）へ変換を行い、変換後データを出力するバッチアプリケーションである。

主な責務は以下の通りである。

* パラメータファイルの読み込みおよび内容チェック
* 変換元文字コードへの変換処理
* 出力ファイルの生成
* 終了コードの返却（正常終了/処理指示パラメータエラー/処理中エラー）

変換可能な文字コードは Java が Charset としてサポートするもの全てである。ただし、本アプリケーションで動作確認済みのコードタイプは以下とする。

* UTF-8
* Shift_JIS (windows-31j)
* EBCDIC (Cp930)
* JIS X 0201
* ISO-2022-JP

== 機能詳細

=== メイン処理

==== 処理契機

以下のコマンドを実行する。

[source,bash]
----
java -jar encoding-batch-1.0.0-jar-with-dependencies.jar ParameterFile.properties
----

ParameterFile.properties : 処理指示パラメータファイルを指定。拡張子は「.properties」とする。文字コードはUTF-8をサポートする。

==== パラメータ（パラメータファイル内容）

以下の項目を key=value 形式で定義する。

.パラメータ一覧
[cols="2,1,5", options="header"]
|===
|パラメータ名 |必須 |内容

|input.file.path
|○
|入力ファイルのフルパス

|output.file.path
|○
|出力ファイルのフルパス

|source.charset.single
|○
|変換元1バイト文字コード +
Java の Charset.forName() が受け付ける名称を指定する。

|source.charset.double
|○
|変換元2バイト文字コード +
Java の Charset.forName() が受け付ける名称を指定する。 +
※ 漢字項目のみ IBM 漢字コードへ変換する場合は「IBMKANJI」を指定する。

|target.charset.single
|○
|変換先1バイト文字コード +
Java の Charset.forName() が受け付ける名称を指定する。

|target.charset.double
|○
|変換先2バイト文字コード +
Java の Charset.forName() が受け付ける名称を指定する。 +
※ 漢字項目のみ IBM 漢字コードへ変換する場合は「IBMKANJI」を指定する。

|file.id
|○
|ファイル識別子 +
FILE_A, FILE_B, FILE_C, FILE_D, FILE_E, FILE_F のいずれかを指定する。
|===

コードタイプは自由入力であり、Java の Charset.forName() が受け付ける名称を指定する。

* UTF-8 を指定する場合 ... UTF-8
* Shift_JIS を指定する場合 ... windows-31j
* EBCDIC を指定する場合 ... Cp930
* JIS X 0201 を指定する場合 ... JIS_X0201
* ISO-2022-JP を指定する場合 ... ISO-2022-JP

.処理指示パラメータファイルの作成例（FILE_C: UTF-8 → JIS混合）
[source,properties]
----
input.file.path=/data/in/sample_input.txt
output.file.path=/data/out/sample_output.txt
source.charset.single=UTF-8
source.charset.double=UTF-8
target.charset.single=JIS_X0201
target.charset.double=ISO-2022-JP
file.id=FILE_C
----

==== 返却値

* int : 終了コード (0/1/2/3/99)

.終了コード一覧
[cols="1,3", options="header"]
|===
|終了コード |説明

|0
|正常終了

|1
|コマンドライン引数エラー

|2
|パラメータ検証エラー

|3
|文字コード変換エラー

|99
|その他の予期しないエラー
|===

==== 処理内容

. コマンドライン引数からパラメータファイルパスを取得する。

. パラメータファイルを読み込み、key=value 形式でパースし、処理指示パラメータを生成する。
.. パラメータのバリデーションを実行する。
.. エラーが存在した場合、エラーメッセージを出力し、終了コード 2（処理指示パラメータエラー）で終了する。
.. パラメータ内容をログ出力する。

. file.id の値に応じて、適切なファイル処理クラスを選択する。
.. FILE_A または FILE_B の場合 → シンプル全体変換処理
.. FILE_C または FILE_D の場合 → 混合文字コード部分変換処理
.. FILE_E または FILE_F の場合 → 可変長レコード変換処理

. 選択したファイル処理クラスの <<_ファイル処理,ファイル処理>> を呼び出し、入力ファイルパスと出力ファイルパスを渡す。
.. 文字コード変換エラーが発生した場合、エラー内容をログ出力し、終了コード 3 で終了する。
.. その他のエラーが発生した場合、エラー内容をログ出力し、終了コード 99 で終了する。

. 終了コード 0 を返却し、処理を終了する。

=== 処理指示パラメータの読み込みとチェック

パラメータファイルの読み込み時に、自動的にバリデーションが実行される。

==== 処理契機

メイン処理からパラメータファイルパスを指定して呼び出される。

==== 処理内容

. プロパティファイルを読み込み、key=value 形式でパースする。

. 必須パラメータのチェックを行う。以下の項目が null または空文字の場合、例外をスローする。
.. input.file.path
.. output.file.path
.. source.charset.single
.. source.charset.double
.. target.charset.single
.. target.charset.double
.. file.id

. file.id の妥当性チェックを行う。
.. file.id が FILE_A ～ FILE_F のいずれかであることを確認する。
.. 不正な値の場合、IllegalArgumentException をスローする。

. 文字コードの妥当性チェックを行う。
.. source.charset.single, source.charset.double, target.charset.single, target.charset.double それぞれについて、Java の Charset.forName() で変換可能かチェックする。
.. 変換できない場合、UnsupportedCharsetException をスローする。

. サポートする文字コードの組み合わせチェックを行う。
.. 以下の文字コードのみサポート対象であることを確認する。
... UTF-8
... ISO-2022-JP
... JIS_X0201
... CP930 (IBM930)
.. 上記以外の文字コードが指定された場合、IllegalArgumentException をスローする。

. パラメータ内容をログ出力する（ファイルID、入出力パス、変換元/先文字コード）。

. 読み込んだパラメータを返却する。

=== ファイル処理

==== パラメータ

* inputPath : 入力ファイルパス（文字列）
* outputPath : 出力ファイルパス（文字列）

==== 処理内容

. 処理開始ログを出力する（ファイルタイプ、入出力パス、変換元/先文字コード）。

. 入力ファイルをオープンする。

. 出力ファイルをオープンする。

. レコード数カウンタ、エラー数カウンタを 0 で初期化する。

. 入力ファイルから1レコードを読み込む（EOF まで繰り返し）。
.. レコード数カウンタをインクリメントする。
.. ファイルタイプに応じた <<_レコード変換処理,レコード変換処理>> を呼び出す。
... 文字コード変換エラーが発生した場合：
.... エラー数カウンタをインクリメントする。
.... エラーログを出力（レコード番号、エラー内容）。
.... CharacterCodingException をスローする。
.. 変換後のレコードを出力ファイルに書き込む。
.. 1000レコードごとに処理中ログを出力する。

. 入力ファイルをクローズする。

. 出力ファイルをフラッシュしてクローズする。

. 処理完了ログを出力する（処理レコード数、エラー数）。

. エラー数が 0 より大きい場合、CharacterCodingException をスローする。

==== レコード読み込み

ファイルタイプと入力文字コードに応じて、レコードの読み込み方法が異なる。

===== 固定長レコード（FILE_A, FILE_B, FILE_C, FILE_D）

====== UTF-8入力の場合（FILE_C, FILE_D）

UTF-8は可変長エンコーディングのため、**文字数ベース**でレコードを読み込む。

. 文字カウンタを 0 で初期化する。

. 指定文字数（380文字）に達するまで、以下を繰り返す。
.. 1バイト読み込む。
.. 先頭バイトからUTF-8文字のバイト数を判定する。
... 0xxxxxxx → 1バイト文字
... 110xxxxx → 2バイト文字
... 1110xxxx → 3バイト文字
... 11110xxx → 4バイト文字
.. 残りのバイト（2〜4バイト目）を読み込む。
.. バッファに追加する。
.. 文字カウンタをインクリメントする。

. バッファのバイト配列を返却する。

====== JIS/EBCDIC入力の場合（FILE_C, FILE_D）

**バイト数ベース**でレコードを読み込む。

. ファイルタイプから取得したレコード長分のバイト配列を用意する。

. 入力ファイルから指定バイト数を読み込む。

. 読み込みバイト数がレコード長と一致しない場合、IOException をスローする（レコード不完全エラー）。

. 読み込んだバイト配列を返却する。

===== 可変長レコード（FILE_E, FILE_F）

可変長レコードは以下の構造を持つ。

----
[0-3バイト] ブロック長（4バイト、ビッグエンディアン）
[4-7バイト] レコード長（4バイト、ビッグエンディアン）
[8バイト以降] データ部分
----

. 先頭4バイトを読み込み、ビッグエンディアンでブロック長を取得する。

. 残りのブロック（ブロック長 - 4 バイト）を読み込む。

. ブロック長バイト配列と残りブロックバイト配列を結合して返却する。

<<<

=== レコード変換処理

レコード変換処理は、ファイルタイプ（FILE_A ～ FILE_F）により処理内容が異なる。

==== FILE_A / FILE_B 処理

===== 概要

1バイト文字のみで構成されるファイルを全体一括変換する。

レコード単位の処理は行わず、ファイル全体を読み込んで変換する。

===== パラメータ

* inputPath : 入力ファイルパス（文字列）
* outputPath : 出力ファイルパス（文字列）

===== 返却値

なし（ファイル出力）

===== 処理内容

. 処理開始ログを出力する（ファイルタイプ、入出力パス、変換元/先文字コード）。

. 入力ファイル全体を読み込む。

. 業務共通クラス CodeConverter の convertCharset() を呼び出し、ファイル全体を変換する。
.. 引数：ファイル全体のバイト配列, sourceCharsetSingle（変換元1バイト文字コード）, targetCharsetSingle（変換先1バイト文字コード）
.. 返却値：変換後のバイト配列

. 変換後のバイト配列をファイルに出力する。

. 処理完了ログを出力する（入力バイト数、出力バイト数）。

<<<

==== FILE_C / FILE_D 処理

===== 概要

380バイト（またはUTF-8の場合380文字）固定長、1バイト文字と2バイト文字（漢字）が混合するファイルを部分変換する。

ファイルは以下のレコード構成を持つ。

. ヘッダーレコード（1バイト目='1'）×1
. データレコード（1バイト目='2'）×N（1つ以上）
. トレーラーレコード（1バイト目='8'）×1
. エンドレコード（1バイト目='9'）×1

**レコード読み込み方法**:

* UTF-8入力の場合: **380文字**を文字数ベースで読み込む（バイト数は可変）
* JIS/EBCDIC入力の場合: **380バイト**をバイト数ベースで読み込む（固定長）

各レコードの先頭1バイト目（または1文字目）でレコード種別を判定し、データレコードの場合は漢字フィールドのみを選択的に変換する。ヘッダー、トレーラー、エンドレコードは全体を1バイト文字として変換する。

===== パラメータ

* record : 入力レコード（380バイトまたは380文字）

===== 返却値

* 変換後のレコード（バイト配列、380バイト）

===== 処理内容

. 変換元文字コードが UTF-8 かどうかを判定する。
.. UTF-8 の場合 → <<_utf8レコード処理,UTF-8レコード処理>> へ
.. JIS/EBCDIC の場合 → <<_バイトベースレコード処理,バイトベースレコード処理>> へ

====== UTF-8レコード処理

. 入力レコードを UTF-8 文字列に変換する。

. 先頭1文字目でレコード種別を判定する。
.. '1' の場合 → ヘッダーレコード → <<_ヘッダーレコード処理,ヘッダーレコード処理>> へ
.. '2' の場合 → データレコード → <<_データレコード処理utf8,データレコード処理（UTF-8）>> へ
.. '8' の場合 → トレーラーレコード → <<_ヘッダーレコード処理,ヘッダーレコード処理>> へ
.. '9' の場合 → エンドレコード → <<_ヘッダーレコード処理,ヘッダーレコード処理>> へ
.. それ以外 → IllegalArgumentException をスローする

====== バイトベースレコード処理

. レコード長が 380 バイトであることを確認する。
.. 不正な場合、IllegalArgumentException をスローする。

. 先頭1バイト目でレコード種別を判定する。
.. 0x31 ('1') の場合 → ヘッダーレコード → <<_ヘッダーレコード処理,ヘッダーレコード処理>> へ
.. 0x32 ('2') の場合 → データレコード → <<_データレコード処理バイトベース,データレコード処理（バイトベース）>> へ
.. 0x38 ('8') の場合 → トレーラーレコード → <<_ヘッダーレコード処理,ヘッダーレコード処理>> へ
.. 0x39 ('9') の場合 → エンドレコード → <<_ヘッダーレコード処理,ヘッダーレコード処理>> へ
.. それ以外 → IllegalArgumentException をスローする

====== ヘッダーレコード処理

ヘッダーレコード、トレーラーレコード、エンドレコードは全体を1バイト文字として変換する。

. 業務共通クラス CodeConverter の convertCharset() を呼び出し、レコード全体を変換する。
.. 引数：record（入力レコード全体）, sourceCharsetSingle, targetCharsetSingle
.. 返却値：変換後のバイト配列

. 変換後のレコードを返却する。

====== データレコード処理（UTF-8）

データレコード内の漢字フィールドを文字位置ベースで抽出し、変換する。

. レコード文字列から2文字目（インデックス1）を取得し、データ種別とする。

. データ種別に応じた漢字フィールド定義（文字位置ベース）を取得する。
.. FILE_C の場合：
... 漢字フィールド1: 50-74文字目（25文字）
... 漢字フィールド2: 150-174文字目（25文字）
... 漢字フィールド3: 300-324文字目（25文字）
.. FILE_D の場合：
... データ種別='1': 漢字フィールド1: 100-124文字目（25文字）、漢字フィールド2: 200-224文字目（25文字）
... データ種別='2': 漢字フィールド1: 120-144文字目（25文字）、漢字フィールド2: 250-274文字目（25文字）

. 現在の文字位置を 0 で初期化する。

. 漢字フィールド定義リストをループ処理する。
.. 現在位置から漢字フィールド開始位置までの部分文字列を抽出する（1バイト文字部分）。
.. 1バイト文字部分を UTF-8 バイト配列に変換する。
.. 業務共通クラス CodeConverter の convertCharset() を呼び出し、1バイト文字を変換する。
... 引数：1バイト文字のバイト配列, sourceCharsetSingle, targetCharsetSingle
... 返却値：変換後のバイト配列
.. 変換後のバイト配列を結果バッファに追加する。
.. 漢字フィールドの部分文字列を抽出する。
.. 漢字フィールドを UTF-8 バイト配列に変換する。
.. 業務共通クラス CodeConverter の convertCharset() を呼び出し、漢字フィールドを変換する。
... 引数：漢字フィールドのバイト配列, sourceCharsetSingle, targetCharsetSingle
... 返却値：変換後のバイト配列
... 備考：UTF-8 には漢字が含まれないため、実際は1バイト文字として変換される
.. 変換後のバイト配列を結果バッファに追加する。
.. 現在位置を漢字フィールド終了位置に更新する。

. 残りの部分文字列（最後の漢字フィールド以降）を抽出し、1バイト文字として変換する。

. 業務共通クラス CodeConverter の convertCharset() を呼び出す。
.. 引数：残り部分のバイト配列, sourceCharsetSingle, targetCharsetSingle
.. 返却値：変換後のバイト配列

. 変換後のバイト配列を結果バッファに追加する。

. 結果バッファのバイト配列を返却する。

====== データレコード処理（バイトベース）

データレコード内の漢字フィールドをバイト位置ベースで抽出し、変換する。

. レコードの2バイト目（インデックス1）を取得し、データ種別とする。

. データ種別に応じた漢字フィールド定義（バイト位置ベース）を取得する。
.. FILE_C の場合：
... 漢字フィールド1: 50-99バイト（50バイト = 25文字）
... 漢字フィールド2: 150-199バイト（50バイト = 25文字）
... 漢字フィールド3: 300-349バイト（50バイト = 25文字）
.. FILE_D の場合：
... データ種別=0x31('1'): 漢字フィールド1: 100-149バイト、漢字フィールド2: 200-249バイト
... データ種別=0x32('2'): 漢字フィールド1: 120-169バイト、漢字フィールド2: 250-299バイト

. 現在のバイト位置を 0 で初期化する。

. 漢字フィールド定義リストをループ処理する。
.. 現在位置から漢字フィールド開始位置までのバイト配列を抽出する（1バイト文字部分）。
.. 業務共通クラス CodeConverter の convertCharset() を呼び出し、1バイト文字を変換する。
... 引数：1バイト文字のバイト配列, sourceCharsetSingle, targetCharsetSingle
... 返却値：変換後のバイト配列
.. 変換後のバイト配列を結果バッファに追加する。
.. 漢字フィールドのバイト配列を抽出する。
.. 業務共通クラス CodeConverter の convertDoubleByteCharset() を呼び出し、漢字フィールドを変換する。
... 引数：漢字フィールドのバイト配列, sourceCharsetDouble, targetCharsetDouble
... 返却値：変換後のバイト配列
.. 変換後のバイト配列を結果バッファに追加する。
.. 現在位置を漢字フィールド終了位置+1に更新する。

. 残りのバイト配列（最後の漢字フィールド以降）を抽出し、1バイト文字として変換する。

. 業務共通クラス CodeConverter の convertCharset() を呼び出す。
.. 引数：残り部分のバイト配列, sourceCharsetSingle, targetCharsetSingle
.. 返却値：変換後のバイト配列

. 変換後のバイト配列を結果バッファに追加する。

. 結果バッファのバイト配列を返却する。

==== FILE_E / FILE_F 処理

===== 概要

可変長レコードを処理し、制御文字変換も実施する。

レコード構造:

----
[0-3バイト] ブロック長（4バイト、ビッグエンディアン）
[4-7バイト] レコード長（4バイト、ビッグエンディアン）
[8バイト以降] データ部分（変換対象）
----

制御文字変換:

* EBCDIC(CP930) → JIS/UTF-8: 0xB4 → 0x74
* JIS/UTF-8 → EBCDIC(CP930): 0x74 → 0xB4

===== パラメータ

* record : 入力レコード（可変長）

===== 返却値

* 変換後のレコード（バイト配列、可変長）

===== 処理内容

. レコード長が最低8バイト（ブロック長4バイト + レコード長4バイト）以上であることを確認する。
.. 不正な場合、IllegalArgumentException をスローする。

. レコードの0-3バイト目からブロック長を読み込む（ビッグエンディアン）。

. レコードの4-7バイト目からレコード長を読み込む（ビッグエンディアン）。

. ログ出力：元のブロック長、レコード長

. データ部分（8バイト目以降）を抽出する。

. 業務共通クラス CodeConverter の convertCharset() を呼び出し、データ部分を変換する。
.. 引数：データ部分のバイト配列, sourceCharsetSingle, targetCharsetSingle
.. 返却値：変換後のバイト配列
.. 備考：convertCharset() 内部で制御文字変換（0xB4 ↔ 0x74）も実施される

. 新しいブロック長を計算する。
.. newBlockLength = 4（ブロック長ヘッダー）+ 4（レコード長ヘッダー）+ 変換後データ長

. 新しいレコード長を計算する。
.. newRecordLength = 4（レコード長ヘッダー）+ 変換後データ長

. ログ出力：変換後のブロック長、レコード長、データ長

. 結果バイト配列を組み立てる。
.. 新しいブロック長（4バイト、ビッグエンディアン）を書き込む
.. 新しいレコード長（4バイト、ビッグエンディアン）を書き込む
.. 変換後のデータを書き込む

. 結果バイト配列を返却する。

== エラー処理

=== エラー発生時の動作

==== パラメータエラー

処理指示パラメータのチェックでエラーが発生した場合：

. エラーログを出力する（エラー内容、パラメータ名）。

. 終了コード 2 を返却し、処理を終了する。

==== 文字コード変換エラー

ファイル処理中に文字コード変換エラーが発生した場合：

. エラーログを出力する（エラー発生レコード番号、エラー内容）。

. 処理を即座に中断する。

. 終了コード 3 を返却し、処理を終了する。

. 出力ファイルは部分的に生成された状態で残る（削除されない）。

予期しないエラーが発生した場合：

. エラーログを出力する（スタックトレース含む）。

. 終了コード 99 を返却し、処理を終了する。

== 補足情報

=== UTF-8 入力時の特殊処理

UTF-8 は可変長エンコーディングのため、固定長ファイル（FILE_C, FILE_D）の処理時は**文字数ベース**でフィールドを抽出し、レコードを読み込む。

==== レコード読み込み

FILE_C/D で UTF-8 を入力とする場合、レコードは**文字数**で区切られる。

* UTF-8 入力: **380文字**で1レコード（バイト数は可変）
* JIS/EBCDIC 入力: **380バイト**で1レコード（固定長）

例:
```
UTF-8レコード: "ABC" + "アイウ" + ... (380文字)
├─ "ABC" = 3バイト
├─ "アイウ" = 9バイト（3文字×3バイト）
└─ ...
合計: 380文字 = 可変バイト数（500〜600バイト程度）

↓ CP930変換

CP930レコード: 380文字 = 380バイト（固定長）
```

==== フィールド抽出

==== 前提条件

* UTF-8 ファイルには漢字が含まれない（英数字・半角カナのみ）
* UTF-8 入力ファイルは 380**文字**で構成される
* JIS/EBCDIC 出力ファイルは 380**バイト**固定長となる

==== 処理フロー

----
UTF-8 入力（380文字、可変バイト長）
    ↓
文字数ベースでレコード読み込み
    ↓
文字位置でフィールド抽出
    ↓
JIS/EBCDIC に変換
    ↓
出力（380バイト固定長）
----

=== ISO-2022-JP のエスケープシーケンス処理

2バイト文字（漢字）の変換時、ISO-2022-JP のエスケープシーケンスは業務共通クラス CodeConverter 内部で自動的に処理される。

==== 変換元が ISO-2022-JP の場合

入力データにはエスケープシーケンスが含まれない想定で、CodeConverter.convertDoubleByteCharset() 内部でエスケープシーケンスを付与して変換を行う。

----
入力：[漢字データ]（ESCシーケンスなし）
    ↓ convertDoubleByteCharset() 内部処理
処理：ESC $ B [漢字データ] ESC ( B
    ↓ 文字コード変換
出力：[変換後データ]
----

==== 変換先が ISO-2022-JP の場合

CodeConverter.convertDoubleByteCharset() 内部で変換後にエスケープシーケンスを除去し、純粋な漢字データのみを返却する。

----
入力：[データ]
    ↓ 文字コード変換
処理：ESC $ B [漢字データ] ESC ( B
    ↓ convertDoubleByteCharset() 内部処理
出力：[漢字データ]（ESCシーケンス除去）
----

=== 制御文字変換（FILE_E, FILE_F）

可変長レコード処理時、業務共通クラス CodeConverter の convertCharset() 内部で制御文字を自動的に変換する。

[cols="2,2,2", options="header"]
|===
|変換方向 |変換元 |変換先

|EBCDIC → JIS/UTF-8
|0xB4 (EBCDIC 'u')
|0x74 (JIS/UTF-8 't')

|JIS/UTF-8 → EBCDIC
|0x74 (JIS/UTF-8 't')
|0xB4 (EBCDIC 'u')
|===
